# L5: Capstone — ADR Architect Agent ⭐

**Day 2 · 17:05–17:50 (45 minutes capstone)**

> The finale. Take a published Copilot skill (`create-architectural-decision-record`) and a published agent (Azure Principal Architect) — **customize both for your team's conventions** — then have the agent scan a Terraform fixture, spot an undocumented decision, and write a real ADR.

---

## 🎯 What You'll Do

1. **Open** the upstream skill + agent (copied into the starter from [awesome-copilot](https://github.com/github/awesome-copilot)).
2. **Customize the skill** — add 1–2 org-specific requirements (front-matter field, required section, naming gate).
3. **Customize the agent** — trim tools, add constraints, point at *your* IaC layout.
4. **Run the agent** — prompt: *"find one undocumented decision in `terraform/` and write an ADR for it through my skill."*

## 🔧 The Lab Flow (45 minutes)

- **Part 1 — Open the starter (≈5 min)** — Browse the upstream skill + agent + Terraform fixture
- **Part 2 — Customize the skill (≈10 min)** — Add 1–2 `TODO (org)` extensions you can defend in a sentence
- **Part 3 — Customize the agent (≈10 min)** — Trim tools, add org constraints, point at `terraform/`
- **Part 4 — Run the agent (≈10 min)** — Prompt → new ADR appears at `docs/adr/adr-NNNN-<slug>.md`
- **Part 5 — Iterate (≈10 min)** — Tighten skill or agent if output isn't right; re-run

## 🚨 Troubleshooting During the Lab

| Issue | Solution |
|-------|----------|
| Agent doesn't list ADRs / can't read files | `.agent.md` `tools:` is opt-OUT. Make sure `'edit/editFiles'` and `'search/codebase'` are in the list (note the namespaced form — bare `codebase` is the old name and VS Code will flag it as renamed) |
| Skill changes don't take effect | Reload the VS Code window after editing skill or agent files |
| Agent writes outside `docs/adr/` | Tighten the agent's constraint block — "Write ADR files ONLY under `docs/adr/`" |
| Agent re-numbers an existing ADR | Agent skipped step 1 (inventory). Reinforce "Read `docs/adr/` FIRST" in Approach |
| Output ignores your `[ORG]` field | Make it `MUST` in the skill's Requirements section, not just an `[ORG]` annotation |

**Still stuck?** Raise your hand. The `solution/` folder shows one customized reference.
